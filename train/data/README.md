# Teacher trajectories for SFT warm-start

**The canonical training input is `seed_combined.jsonl`** (10 episodes, 82 steps, 78 trainable after empty-prompt filter). It merges two teachers with deliberately different characteristics:

| Source | Episodes | Mean score | Resolved | Role |
|---|---|---|---|---|
| Claude Opus 4.7 via pool server | 6 | 0.769 | 6/6 | **Expert demos.** Hand-driven by Claude Opus acting as the agent, observation-only (no runbook), recorded 2026-04-24. |
| Llama-3.3-70B via Fireworks API | 4 | 0.725 | 3/4 | **Realistic agent.** Generated by `collect_trajectories.py --driver fireworks`, 2026-04-25. Noisier, occasionally picks wrong rollback target — useful variance for GRPO. |

## Files

- `seed_combined.jsonl` — merged training corpus (canonical input for `sanity_run.ipynb`)
- `claude_seed.jsonl` — 6 Claude episodes only (kept for provenance)
- `llama33_70b_smoke4.jsonl` — 4 Llama-3.3-70B episodes only
- `claude_<scenario>.jsonl` — raw per-episode event logs from the Claude run (reset/step/evaluate events, auditable)

## Scenario coverage

| Scenario | Teacher episodes |
|---|---|
| worker_deploy_cascade | Claude ×1, Claude procgen variant ×1 |
| db_config_rollout | Claude ×1, Claude procgen ×1, Llama ×1 |
| gateway_auth_rollout | Claude ×1, Claude procgen ×1, Llama ×1 |
| payment_webhook_misconfig | Llama ×1 |
| cache_stale_state | Llama ×1 |
| schema_drift_missing_migration | **0 — gap to fill** |

## Gotchas

- **Filter `len(prompt) < 50`**: 4 of the Claude rollback steps lost their prior observation to a chained-call logging bug in the driver script. Reference loader implementation is in `train/sanity_run.ipynb` cell 10.
- **Fireworks rate limits**: the free-tier daily quota on `accounts/fireworks/models/llama-v3p3-70b-instruct` is tight enough that `--parallelism > 2` trips 429s that cascade to the heuristic fallback. Use `--parallelism 2` and the driver's built-in `Retry-After` backoff for any rerun.
- **Procgen variants**: included in the Claude set but not yet in the Llama set. Once Fireworks' daily quota resets, rerun `collect_trajectories.py --driver fireworks --scenarios all --episodes-per-model 60 --parallelism 2` to fill in the remaining variants.

## Reproduce

```bash
# 1. Boot env (local uvicorn or live HF Space)
python -m uvicorn unified_incident_env.server.app:create_compatible_app --factory --port 8000

# 2. Collect Llama-3.3-70B teacher episodes via Fireworks
export FIREWORKS_API_KEY=...
python train/collect_trajectories.py \
  --env-url http://127.0.0.1:8000 \
  --scenarios all \
  --models accounts/fireworks/models/llama-v3p3-70b-instruct \
  --episodes-per-model 60 \
  --parallelism 2 \
  --driver fireworks \
  --output train/data/llama33_70b_60.jsonl

# 3. Re-merge (deterministic)
cat train/data/claude_seed.jsonl train/data/llama33_70b_*.jsonl > train/data/seed_combined.jsonl
```
